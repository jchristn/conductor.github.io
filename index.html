Why Conductor?

Without Conductor

  • Hardcoded endpoints scattered across services
  • One provider goes down, your app goes down
  • No visibility into request routing or failures
  • Switching providers means rewriting integration code
  • Every team manages their own model configs

With Conductor

  • One unified endpoint for all model providers
  • Automatic failover routes around unhealthy endpoints
  • Full request history, health monitoring, and a dashboard
  • Swap providers by changing config, not code
  • Centralized configuration pinning across tenants

How It Works

Conductor sits between your application and your model endpoints, orchestrating every request.

Load Balancing

Round-robin, random, or first-available routing with weighted distribution. Traffic spreads evenly or by priority.

Health Checking

Background monitoring with configurable thresholds. Unhealthy endpoints drop from rotation, restored automatically on recovery.

Session Affinity

Pin clients to specific endpoints by IP, API key, or custom header. Conversations stay on the same model runner.

Built for Your Role

For Platform Teams

Centralized model management

Define virtual model runners that abstract your backend endpoints. Assign tenants, pin configurations, and enforce consistent settings across your organization — all from one control plane.

  • Virtual model runners with multi-endpoint routing
  • Full multi-tenancy with RBAC (global admin, tenant admin, user)
  • Configuration pinning for temperature, top_p, and more
  • 4 database backends: SQLite, PostgreSQL, MySQL, SQL Server

For AI Engineers

You already know the API

Conductor speaks OpenAI, vLLM, Gemini, and Ollama. Point existing SDKs and clients at Conductor to gain routing, balancing, and failover without rewriting your application architecture.

Python 
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:9000/vmr/my-runner/v1",
    api_key="default"
)

response = client.chat.completions.create(
    model="llama3.2",
    messages=[{"role": "user", "content": "Hello!"}]
)

The First Rehearsal

Up and running in three steps with Docker Compose.

1

Download the compose file

Shell 
curl -o compose.yaml \
  https://raw.githubusercontent.com/jchristn/Conductor/main/docker/compose.yaml
2

Start the containers

Shell 
docker compose up -d
3

Verify it's running

Shell 
curl http://localhost:9000/

Next: Open the Dashboard at localhost:9100

Docker Run
Shell 
# Start Conductor server
docker run -d --name conductor \
  -p 9000:8080 \
  jchristn77/conductor:latest

# Start Conductor UI
docker run -d --name conductor-ui \
  -p 9100:8080 \
  jchristn77/conductorui:latest
From Source
Shell 
git clone https://github.com/jchristn/Conductor.git
cd Conductor/src/Conductor
dotnet build
dotnet run
